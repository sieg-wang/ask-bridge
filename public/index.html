給 AI Agent 用的研究橋接器

把終端機，
接上 ChatGPT 與 Gemini

ask-bridge 以 Rust 驅動真實 Chrome，自動把 prompt 送進網站型 AI，再把回覆取回你的終端機。讓主要 Coding Agent 專注在高價值工作，把探索性研究交給網站額度。

Rust 的所有權模型：每個值有唯一所有者，離開作用域即自動釋放⋯（回覆自動取回終端機）

不是取代，而是橋接

開發過程中有大量探索性 AI 需求——查資料、整理文件、比較方案、摘要錯誤、產生初稿。這些任務未必需要主要 Coding Agent 親自做，也不值得消耗與改程式、跑測試相同的額度。

  • 額度分開計算

    網站型 AI 的使用額度與 Coding Agent 執行額度通常獨立。把研究工作委外，讓高價值額度留給程式碼。

  • 不離開工作流程

    不必切換瀏覽器、貼上 prompt、等待、再複製回來。一行指令完成終端機與網站型 AI 的往返。

  • 真實瀏覽器，持久登入

    在真實 Chrome 中執行，只需登入一次。可處理 CAPTCHA、Cloudflare，像一般使用者存取網站功能。

一行指令，完整橋接

多 Provider 切換

以 --provider 在 ChatGPT 與 Gemini 之間切換，或在設定檔指定預設 provider。

ask-bridge --provider gemini "比較這兩個方案"

Pipe 與 stdin

把檔案內容或管線文字直接餵進 prompt，無需離開終端機。

cat src/main.rs | ask-bridge "有記憶體洩漏嗎？"

圖片與文件附件

--image 附上圖片、--file 附上 PDF／Word／CSV／程式碼等文件。

ask-bridge "摘要這份 PDF" --file report.pdf

模型切換

送出前以 --model 切換 ChatGPT 的 GPT-5.4、o3 或 Gemini 的 3.1 Pro。

ask-bridge "證明這題" --model o3

智慧分頁 · 安靜預設

重用、聚焦或開新分頁避免分頁氾濫；預設只輸出回覆，--verbose 顯示完整流程。

ask-bridge "誰是保哥？" --new --verbose

三分鐘上手

1 · 登入

首次執行登入所選 provider，只需一次。

ask-bridge login
ask-bridge --provider gemini login
2 · 直接提問

把 prompt 作為參數傳入，回覆輸出到終端機。

ask-bridge "Rust struct 和 tuple 有何差異？"
3 · 管線與檔案

透過 pipe 餵入內容，或以 --file 附件上傳。

cat src/main.rs | ask-bridge "這段有風險嗎？"
ask-bridge "摘要這份 PDF" --file report.pdf
4 · 切換模型

送出前指定模型或思考強度。

ask-bridge "用幾句話介紹 Rust" --model GPT-5.4
ask-bridge --provider gemini "翻譯這段" --model "3.1 Pro"

從指令到回覆，四個步驟

  1. 01

    瀏覽器初始化

    檢查 Chrome 是否監聽 debug port 9223，若無則以專屬 profile 目錄啟動背景 Chrome。

  2. 02

    MCP Bridge 設定

    自動寫入 mcp_servers.json，設定 chrome-devtools-mcp server 指向本機 9223。

  3. 03

    Client 呼叫

    透過內建 mcp-cli 呼叫 list_pages、select_page、type_text、evaluate_script 等工具。

  4. 04

    狀態輪詢

    以 JavaScript 檢查送出／停止按鈕狀態，擷取回覆元素文字並輸出到 stdout。

馬上開始

前置需求

需要 Node.js 20.19.0 LTS 以上或更新的 LTS 版本，且 node 與 npx 必須可在目前 shell 的 PATH 中執行。ask-bridge 會透過 npx 啟動 chrome-devtools-mcp@latest；Node.js 版本過舊時，MCP server 可能在 initialize 階段直接退出。

node -v
npx -v

macOS

可使用 Homebrew 或 nvm 安裝 Node.js LTS。若使用 nvm，請確認執行 ask-bridge 的同一個 shell 已載入 nvm。Chrome 預設偵測 /Applications/Google Chrome.app/Contents/MacOS/Google Chrome。

Windows

可使用官方安裝程式、winget 或 nvm-windows 安裝 Node.js LTS。安裝後請重新開啟 PowerShell，並確認 node 與 npx 可執行。Chrome 會偵測標準安裝路徑。

Linux

發行版內建套件庫可能提供較舊的 Node.js；建議使用 NodeSource、nvm 或官方 LTS 來源。請安裝 Google Chrome Stable，並確認 google-chrome 或 google-chrome-stable 位於 PATH。

git clone https://github.com/doggy8088/ask-bridge.git
cd ask-bridge && make install
